Important integration note:
- This proposal does not update ShowtimeSeatInventory.Status to Reserved.
- It prevents duplicate active holds by checking BookingHolds/BookingHoldSeats inside a Serializable transaction.
- If the seat-map API must display held seats as Reserved, sync with Dev 1 first because ShowtimeSeatInventory belongs to Dev 1 ownership.
